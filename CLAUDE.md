# CLAUDE.md

## Verification Over Recall

- Never rely on remembered knowledge for libraries, APIs, or frameworks —
  check installed versions and docs before writing code or making claims
- If you lack verified information, acknowledge uncertainty and investigate
  first rather than speculate

## Code Reuse

When adding a feature that requires logic that already exists elsewhere in the codebase, extract it into a shared module — don't copy it. Duplication is not a "minimal change"; it creates two places to maintain identical behaviour.

## Collaboration Style

- Be objective, not agreeable — act as a partner, not a sycophant
- Push back when you disagree, flag tradeoffs honestly, don't sugarcoat problems
- Keep explanations brief and to the point
- Accuracy over speed

## UI Verification

**Always verify UI changes in a real browser before reporting them done.** Type-checking and build success do not prove that a page renders, that an island hydrates, or that user flows work. Use `playwright-cli` (installed at `/opt/homebrew/bin/playwright-cli`) — a terminal wrapper around the Playwright MCP server, drivable from Bash.

Minimal flow:

```bash
npm run dev > /tmp/dev.log 2>&1 &       # background dev server
sleep 4 && grep -E "Local" /tmp/dev.log  # find the port (may not be 4321 if busy)
playwright-cli open http://localhost:<port>/hdx-cod-ab-geocoder/
playwright-cli snapshot                   # accessibility-tree snapshot of current page
playwright-cli eval "() => ({ ... })"     # arbitrary JS in page context
playwright-cli click <ref>                # ref comes from snapshot
playwright-cli close                      # always close at the end
```

Notes:

- `playwright-cli snapshot` writes a YAML accessibility tree to `.playwright-cli/page-*.yml`. The tree includes element `ref` IDs (e.g. `ref=e9`) you pass to `click`/`hover`/`fill`.
- Do NOT write standalone `.spec.ts` files — drive Playwright from Bash via `playwright-cli`. The project has no Playwright test runner configured.
- MapLibre GL renders to a `<canvas>` — use `eval` to check map state (`map.isStyleLoaded()`, `map.getStyle().sources`, etc.) rather than snapshot visibility.
- DuckDB and service worker state are inspectable via `eval`: `navigator.serviceWorker.getRegistrations()`, `navigator.storage.getDirectory()`.

## Architecture

**HDX COD-AB Geocoder** is a browser-only geocoder for OCHA Common Operational Dataset administrative boundaries. No server, no Google Maps API, no auth. Converts lat/lon coordinates to UN P-codes (ADM0–ADM4).

**Stack:** Astro (static site, GitHub Pages) + Svelte 5 (runes) + DuckDB WASM + MapLibre GL + pmtiles protocol

**Deployment:** `https://ocha-dap.github.io/hdx-cod-ab-geocoder` via `.github/workflows/deploy.yml`

### Data sources

| Use | Format | URL |
|-----|--------|-----|
| Country list | Static JSON | `/hdx-cod-ab-geocoder/catalog.json` (pre-built at build time) |
| Map display | PMTiles | `https://data.source.coop/hdx/cod-ab/original/{country}/{id}/{id}.pmtiles` |
| Batch geocoding | GeoParquet | Same base, `.parquet` extension |

`public/catalog.json` is generated by `scripts/gen-catalog.mjs` from the local Portolan STAC catalog at `../hdx-scraper-cod-ab-global/tmp/original`. Run `npm run catalog` after updating the source data.

### Lookup strategy

| Operation | Method |
|-----------|--------|
| Single point (map click or lat/lon input) | `queryRenderedFeatures` on PMTiles — instant, full attributes already in tiles |
| Batch CSV geocoding | DuckDB spatial join on GeoParquet (`ST_Within`); result cached in OPFS |
| XLSForm / XLSX generation | DuckDB attribute-only query (no geometry fetch) → SheetJS |

### Key design decisions

- **GeoParquet + spatial extension → `stoi` bug** — DuckDB WASM's spatial extension hooks into `read_parquet` to parse the GeoParquet `"geo"` key-value metadata. A field in that JSON causes `std::stoi` to throw `"no conversion"`. Fix: fetch the parquet via browser `fetch`, call `removeGeoMetaKey` (in `src/lib/db/geoparquet.ts`) to rename the `"geo"` key in the Thrift compact footer to `"___"`, then register with `db.registerFileBuffer`. Use `registerParquetBuffer(url, name)` from that module wherever `read_parquet(remoteUrl)` would otherwise be used. After the metadata strip the spatial extension still decodes the geometry column as `GEOMETRY` (not `BLOB`), so use `b.geometry` directly in `ST_Within` — not `ST_GeomFromWKB(b.geometry)`.
- **PMTiles carry full attributes** — no need to fetch Parquet for single-point lookup. `queryRenderedFeatures` returns all pcode/name columns directly.
- **Fly-then-query for manual lat/lon input** — `map.flyTo()` then `map.once('idle', ...)` ensures the correct PMTiles tiles are loaded before `queryRenderedFeatures` runs.
- **Highlight via filter** — highlight layers use `map.setFilter` with the matched pcode; no re-fetch needed.
- **No SW asset caching** — `sw.ts` is stripped to PWA installability only. Content-hashed filenames + HTTP cache handle updates. New SW activates only when the user explicitly clicks "Update" (posts `SKIP_WAITING`).
- **Catalog pre-built at build time** — avoids 550+ HTTP requests at runtime. 260KB JSON for 109 countries.
- **Svelte 5 runes** — uses `$state()`, `$effect()`, `$bindable()` — not legacy reactivity.
- **Path alias:** `$lib` resolves to `src/lib/`.
